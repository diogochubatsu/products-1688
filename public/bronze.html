Destination

Product Table — Migration Target

🎯 What migrates to your database 🔒 Scraping data stays here

Clean separation: scraping operational data stays with the scraper, product data migrates to your destination database. This table has only what a Brazilian seller needs to make purchasing decisions.

✓ Migrates (9 fields)
  • offer_id — 1688 product ID (unique)
  • title_cn — original Chinese title
  • title_br — Portuguese translation
  • price_cny — price in CNY
  • price_brl — price in BRL
  • category — product category
  • image_url — product image
  • source_url — 1688 or Rakumart link
  • supplier_name — who sells it
✗ Stays with scraper (operational)
  • taxonomy.n1-n4 — our internal classification
  • match_score — crosswalk confidence
  • enriched_at — last scrape timestamp
  • shop / shop_name — changes frequently
  • bronze_refs — pipeline traceability
  • category (flat) — our mapping

Product Table Schema (SQL)

CREATE TABLE products ( id SERIAL PRIMARY KEY, offer_id BIGINT UNIQUE NOT NULL, -- 1688 product ID title_cn TEXT NOT NULL, -- original Chinese title title_br TEXT, -- Portuguese translation price_cny DECIMAL(10,2), -- price in CNY price_brl DECIMAL(10,2), -- price in BRL category VARCHAR(50), -- product category image_url TEXT, -- product image source_url TEXT, -- 1688 or Rakumart link supplier_name VARCHAR(200), -- who sells it last_updated TIMESTAMP DEFAULT NOW(), -- when we last scraped created_at TIMESTAMP DEFAULT NOW() -- first seen ); -- Indexes for common queries CREATE INDEX idx_products_category ON products(category); CREATE INDEX idx_products_price_brl ON products(price_brl); CREATE INDEX idx_products_offer_id ON products(offer_id);

                          │
                    ┌─────┴─────┐
                    │ Scraping  │

                    │ (stays)   │
                    └───────────┘

Scraping flow:
  1. MTOP discovers products (title, price, image, shop)
  2. SU Detail enriches (supplier company name)
  3. Rakumart crosswalks (Portuguese title, BRL price)
  4. Taxonomy classifies (N1→N4)
  5. Export: clean product table → your database
  6. Operational data stays here for re-scraping

Example Migration Query

-- From silver offers to product table INSERT INTO products (offer_id, title_cn, title_br, price_cny, price_brl, category, image_url, source_url, supplier_name) SELECT o.offer_id, o.mtop->>'title' as title_cn, o.rakumart->>'title_br' as title_br, (o.mtop->>'price_cny')::decimal as price_cny, (o.rakumart->>'price_brl')::decimal as price_brl, o.category, o.mtop->>'image_url' as image_url, o.rakumart->>'url' as source_url, o.su_detail->>'shop_name' as supplier_name FROM silver_offers o WHERE o.rakumart IS NOT NULL -- only matched products ON CONFLICT (offer_id) DO UPDATE SET price_brl = EXCLUDED.price_brl, last_updated = NOW();
