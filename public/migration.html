Product Database — Migration Schema

Target schema for the agent creating the destination product database

Overview

This is the product table that receives data from the scraper. The scraper handles all operational complexity (taxonomy, scoring, crosswalking). This table contains only what a Brazilian seller needs to make purchasing decisions.

Source: Silver offers from /mnt/ssd/1688-only/data/silver/offers/*.json
Destination: PostgreSQL database (your choice of hosting)

SQL DDL

CREATE TABLE products ( id SERIAL PRIMARY KEY, offer_id BIGINT UNIQUE NOT NULL, title_cn TEXT NOT NULL, title_br TEXT, price_cny DECIMAL(10,2), price_brl DECIMAL(10,2), category VARCHAR(50), image_url TEXT, source_url TEXT, supplier_name VARCHAR(200), last_updated TIMESTAMP DEFAULT NOW(), created_at TIMESTAMP DEFAULT NOW() ); -- Indexes for common queries CREATE INDEX idx_products_category ON products(category); CREATE INDEX idx_products_price_brl ON products(price_brl); CREATE INDEX idx_products_supplier ON products(supplier_name); CREATE INDEX idx_products_last_updated ON products(last_updated);

Migration Query

Insert from silver JSON files into the product table. Uses PostgreSQL JSON operators.

-- Insert products from silver offers -- Run from scraper machine or adapt for your ETL INSERT INTO products (offer_id, title_cn, title_br, price_cny, price_brl, category, image_url, source_url, supplier_name) SELECT (o->>'offer_id')::bigint, o->'mtop'->>'title', o->'rakumart'->>'title_br', (o->'mtop'->>'price_cny')::decimal, (o->'rakumart'->>'price_brl')::decimal, o->>'category', o->'mtop'->>'image_url', o->'rakumart'->>'url', o->'su_detail'->>'shop_name' FROM jsonb_array_elements( -- Load from JSON file or view '[{"offer_id":123,"mtop":{"title":"..."},...}]'::jsonb ) AS o WHERE o->'rakumart' IS NOT NULL -- only matched products ON CONFLICT (offer_id) DO UPDATE SET price_brl = EXCLUDED.price_brl, title_br = EXCLUDED.title_br, source_url = EXCLUDED.source_url, last_updated = NOW();

Useful Queries

-- Products by category with margin calc SELECT category, COUNT(*), AVG(price_brl), AVG(price_cny) FROM products WHERE price_brl IS NOT NULL GROUP BY category ORDER BY COUNT(*) DESC; -- Top products by price margin SELECT title_br, price_cny, price_brl, ROUND(price_brl / (price_cny * 0.8), 1) AS markup FROM products WHERE price_cny > 0 AND price_brl > 0 ORDER BY markup DESC LIMIT 20; -- Recently updated products SELECT offer_id, title_br, price_brl, last_updated FROM products ORDER BY last_updated DESC LIMIT 50;

Notes

  • title_br, price_brl, source_url are NULL for unmatched products (no Rakumart crosswalk)
  • supplier_name may be NULL if SU Detail scrape failed
  • image_url may break if alicdn.com changes CDN rules — consider caching locally
  • offer_id is the join key if you need to re-fetch data from the scraper
  • Use ON CONFLICT for upserts — scraper may re-scrape the same product
